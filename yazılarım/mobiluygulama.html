Android Nedir ve Neden Tercih Edilir?

Android, Google tarafından geliştirilen açık kaynaklı mobil işletim sistemidir. Dünya genelinde %70'in üzerinde pazar payına sahip olan Android, milyarlarca cihazda kullanılmaktadır. Android geliştirme, çok geniş bir kullanıcı kitlesine ulaşma imkanı sağlar.

Android uygulamaları Java veya Kotlin programlama dilleri ile geliştirilebilir. Google, 2017'den beri Kotlin'i tercih ettiğini açıkladı ve 2019'da Kotlin'i "Kotlin-first" yaklaşımıyla resmi dil ilan etti. Modern Android geliştirmede Kotlin kullanımı önerilmektedir.

💡 Pro İpucu: 2025 yılında yeni Android uygulamalar için API Level 35 (Android 15) hedeflenmesi zorunludur. Mevcut uygulamalar için de API Level 34 minimum gereklidir.
Android'in Avantajları:
  • • Geniş kullanıcı kitlesi (%70+ pazar payı)
  • • Açık kaynak ve esnek platform
  • • Google Play Store ile kolay dağıtım
  • • Güçlü geliştirme araçları (Android Studio)
  • • Zengin kütüphane ve framework desteği
Pazar Durumu
Android geliştiriciler için büyük bir iş fırsatı alanıdır. Mobil geliştirici maaşları diğer yazılım geliştirme alanlarına göre %10-15 daha yüksektir.

Google, Netflix, Uber, WhatsApp gibi büyük şirketler Android platformunu tercih etmektedir. Android'in Linux tabanlı açık kaynak yapısı, geliştiricilere büyük esneklik sağlar ve çeşitli cihaz türlerinde çalışabilir.

Android Studio Kurulumu ve İlk Proje

Android Studio, Google tarafından geliştirilen resmi Android geliştirme ortamıdır. IntelliJ IDEA tabanlı olan bu IDE, Android geliştirme için özel olarak optimize edilmiştir ve 2025 sürümü ile birlikte yapay zeka destekli kod tamamlama özelliği gelmiştir.

Kurulum Adımları

2. Kurulum sihirbazını takip edin
3. SDK Manager'dan gerekli API seviyelerini indirin

Kotlin Programlama Temelleri

Kotlin, JetBrains tarafından geliştirilen ve Google tarafından Android için tercih edilen modern bir programlama dilidir. Java ile %100 uyumlu olan Kotlin, daha az kod yazmayı sağlar ve null güvenliği gibi modern özellikler sunar. 

// Değişken tanımlama
val name = "Android"        // Değiştirilemez (immutable)
var version = 15           // Değiştirilebilir (mutable)

// Null güvenliği
var nullableString: String? = null
val length = nullableString?.length ?: 0

// Fonksiyon tanımlama
fun greetUser(name: String): String {
    return "Merhaba, $name!"
}

// Tek satır fonksiyon
fun add(a: Int, b: Int) = a + b
💡 Kotlin vs Java: Kotlin %40 daha az kod yazmayı sağlar. Null Pointer Exception'ları büyük ölçüde ortadan kaldırır ve lambda ifadeleri ile fonksiyonel programlamayı destekler.

  • Null Safety: NullPointerException'ları önler
  • Extension Functions: Mevcut sınıflara fonksiyon ekler
  • Data Classes: Otomatik kod üretimi
  • Coroutines: Asenkron programlama desteği
  • Smart Cast: Otomatik tip dönüşümü
Coroutines (Eş zamanlı İşlemler)
Kotlin Coroutines, arka plan işlemlerini yönetmenin modern yoludur. Thread'lere göre daha hafif ve verimlidir.

Jetpack Compose, Android için Google'ın modern kullanıcı arayüzü geliştirme toolkit'idir. XML layout dosyalarını tamamen ortadan kaldırarak, UI'ı Kotlin kod ile tanımlamanızı sağlar. 2025'te Android uygulamalarının %60'ı Compose kullanmaktadır.

Compose'un Avantajları: Daha az kod, daha güvenli, güçlü araçlar ve hızlı geliştirme. XML yerine tamamen Kotlin ile UI oluşturabilirsiniz.

Veri Yönetimi ve Room Database

Modern Android uygulamalarında veri yönetimi için Room Database kullanılır. Room, SQLite üzerine inşa edilmiş güçlü bir ORM (Object Relational Mapping) kütüphanesidir ve compile-time verification sağlar. 

// Entity (Veri modeli)
@Entity(tableName = "users")
data class User(
    @PrimaryKey val id: String,
    val name: String,
    val email: String,
    val createdAt: Long = System.currentTimeMillis()
)

// DAO (Data Access Object)
@Dao
interface UserDao {
    
    @Query("SELECT * FROM users ORDER BY createdAt DESC")
    fun getAllUsers(): Flow<List<User>>
    
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertUser(user: User)
    
    @Delete
    suspend fun deleteUser(user: User)
    
    @Query("SELECT * FROM users WHERE name LIKE :searchQuery")
    fun searchUsers(searchQuery: String): Flow<List<User>>
}
// Database sınıfı
@Database(
    entities = [User::class],
    version = 1,
    exportSchema = false
)
@TypeConverters(Converters::class)
abstract class AppDatabase : RoomDatabase() {
    
    abstract fun userDao(): UserDao
    
    companion object {
        @Volatile
        private var INSTANCE: AppDatabase? = null
        
        fun getDatabase(context: Context): AppDatabase {
            return INSTANCE ?: synchronized(this) {
                val instance = Room.databaseBuilder(
                    context.applicationContext,
                    AppDatabase::class.java,
                    "app_database"
                ).build()
                INSTANCE = instance
                instance
            }
        }
    }
}
💡 Repository Pattern: DAO'ları doğrudan kullanmak yerine Repository sınıfları oluşturun. Bu sayede veri kaynaklarını soyutlayabilir ve test edilebilirliği artırabilirsiniz.
Room'un Avantajları:
  • Compile-time verification: SQL sorguları derleme zamanında kontrol edilir
  • Flow desteği: Reactive programming için Flow entegrasyonu
  • Migration desteği: Veritabanı şeması güncellemeleri
  • Coroutine desteği: Suspend fonksiyonlar ile async işlemler
DataStore (SharedPreferences'ın Yerine)
Basit key-value verileri için Room yerine Jetpack DataStore kullanın. Proto DataStore daha güvenli, Preferences DataStore daha basittir. 
// Repository sınıfı örneği

Network İşlemleri ve API Entegrasyonu

Android uygulamalarında API çağrıları için Retrofit kütüphanesi endüstri standardıdır. Retrofit, HTTP isteklerini Kotlin interface'leri ile kolayca yönetmenizi sağlar ve coroutine desteği sunar. 

// API interface tanımı
interface ApiService {
    
    @GET("users")
    suspend fun getUsers(): Response<List<User>>
    
    @GET("users/{id}")
    suspend fun getUserById(
        @Path("id") userId: String
    ): Response<User>
    
    @POST("users")
    suspend fun createUser(
        @Body user: CreateUserRequest
    ): Response<User>
    
    @PUT("users/{id}")
    suspend fun updateUser(
        @Path("id") userId: String,
        @Body user: UpdateUserRequest
    ): Response<User>
}
// Retrofit instance oluşturma
object ApiClient {
    private const val BASE_URL = "https://api.example.com/"
    
    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(HttpLoggingInterceptor().apply {
            level = HttpLoggingInterceptor.Level.BODY
        })
        .addInterceptor { chain ->
            val request = chain.request().newBuilder()
                .addHeader("Authorization", "Bearer your-token")
                .build()
            chain.proceed(request)
        }
        .connectTimeout(30, TimeUnit.SECONDS)
        .readTimeout(30, TimeUnit.SECONDS)
        .build()
    
    val apiService: ApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}
💡 Offline-First Yaklaşım: NetworkBoundResource pattern kullanarak önce yerel veritabanından veri gösterin, sonra API'den güncel veriyi çekin ve veritabanını güncelleyin.
Network Güvenliği:
  • HTTPS: Her zaman güvenli bağlantı kullanın
  • Certificate Pinning: SSL sertifikası doğrulama
  • Token Yönetimi: JWT token'ları güvenli saklayın
  • Request/Response Şifreleme: Hassas veriler için
Hata Yönetimi
Network hatalarını try-catch blokları ile yakalayın. Timeout, network unavailable gibi durumları kullanıcıya anlamlı mesajlarla bildirin. 
// Repository'de network çağrısı
class UserRepository(
    private val apiService: ApiService,
    private val userDao: UserDao
) {
    suspend fun getUsers(): Resource<List<User>> {
        return try {
            val response = apiService.getUsers()
            if (response.isSuccessful) {
                response.body()?.let { users ->
                    userDao.insertUsers(users)
                    Resource.Success(users)
                } ?: Resource.Error("Veri bulunamadı")
            } else {
                Resource.Error("Sunucu hatası: ${response.code()}")
            }
        } catch (e: Exception) {
            Resource.Error(e.message ?: "Bilinmeyen hata")
        }
    }
}
Dikkat: Network çağrıları asla Main Thread'de yapılmamalıdır. Coroutines veya Background Thread kullanın. ANR (Application Not Responding) hatasına neden olabilir.

Performans Alanları:
  • Memory: Memory leak'leri önleme
  • CPU: Gereksiz işlemleri azaltma
  • Network: Veri kullanımını optimize etme
  • Battery: Güç tüketimini minimize etme
  • UI: 60 FPS smooth rendering
Android Studio Profiler: Memory Profiler, CPU Profiler ve Network Profiler kullanarak uygulamanızın performansını sürekli izleyin ve bottleneck'leri tespit edin.

}

Google Play Store'a Yayınlama

2025 yılında Google Play Store'a uygulama yayınlamak için App Bundle (AAB) formatı zorunludur. AAB formatı, APK'ya göre %15 daha küçük indirme boyutu sağlar ve Play Dynamic Delivery ile optimize dağıtım sunar.

2025 Gereksinimleri:
  • Target API Level 35: Yeni uygulamalar için zorunlu
  • App Bundle: AAB formatı zorunlu
  • 64-bit desteği: ARM64 ve x86-64 gerekli
  • Privacy Policy: Kullanıcı verisi toplayan uygulamalar için
  • Content Rating: IARC rating sistemi
